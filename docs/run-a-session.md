# Run a Session

Use this workflow during play.

## Open the Live Dashboard

Run **D&D Hub: Start Session**.

The Session Run Dashboard is the main live control surface. It keeps the current scene, linked map, encounter, music, sound effects, handouts, and party state in one place.

![Session Run Dashboard](assets/usability/session-run-dashboard.png)

## Work Scene by Scene

Use the scene controls to move through prepared scenes. When a scene has links, the dashboard exposes direct actions:

| Link | Live action |
| --- | --- |
| Map | Open the GM map view |
| Encounter | Start or load the encounter |
| Music | Play scene music through the music player |
| SFX | Trigger sound effects |
| Handout | Open projection controls |

When a link is missing, the dashboard shows a prompt instead of leaving the area blank.

## Use Inline Controls

Session notes can include small inline controls for table use:

| Control | Use |
| --- | --- |
| Sound effect | Trigger an SFX from inside text |
| Scene music | Start linked scene music |
| Encounter | Start a linked encounter |
| Handout | Open handout projection controls |
| Map | Open a linked map |

Inline controls are meant for moments where the cue belongs directly in the prose.

## End the Session

Keep the session note as the session record. Add outcomes, unresolved questions, loot, NPC reactions, and next hooks before closing the session.
